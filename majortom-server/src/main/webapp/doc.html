1. TMQL/JSON over HTTP API

Der MaJorToM Server unterstützt eine über URL gesteuerte API.

Der neue Server unterstützt nun ausschließlich TMQL queries und gibt ein JSON Objekt zurück in der Form:

{code : "0", msg : "OK", data : [] }

code ist entweder 0 oder 1. Eine erfolgreiche Abfrage hat den Code 0, eine fehlgeschlagene den Code 0. msg Enthält eine Nachricht für den Nutzer. Ist die Abfrage erfolgreich ist die Nachrich "OK", ansonsten enthält sie eine Fehlermeldung. data Enthält die Werte

Unter http://majortom.topicmapslab.de/majortom-server-ui/ findet man ein Webinterface um TMQL Queries zu testen. Einige Beispiele sind bereits vorhanden und können via Combobox ausgewählt und ausgeführt werden.

1.1. ID einer Topic Map erhalten

Jede Topic Map hat eine Id. Die Id ist die MD5 Summe des base locators.

Die URL ist der base locator der Topic Map und entspricht der URL für die Topic-Map-Auswahl im Webinterface.

1.2. TMQL queries abwerfen

Der Query kann auch als POST-Parameter übergeben werden. Der Name des Parameters ist query.

1.3. TMQL

TMQL ist die Abfragesprache für Topic Maps. Eine ausführliche Dokumentation inklusive Beispielen findet man unter http://docs.topicmapslab.de/tmql4j/ .

1.4. Das Webinterface

Das Webinterface ist eine eigenständige Annwendung welche den tmql-Endpoint des Majortom-Servers anspricht und die Ergebnisse grob auswertet. Zusätzlich enthält es Beispiele für TMQL-Queries.

Im folgenden werden die Elemente des Formulars beschrieben.

  • Server Diese Auswahl-Box dient zur Auswahl des Servers auf dem die Topic map geladen ist. Im Moment gibt es lediglich den Server http://majortom.topicmapslab.de/majortom-server/

  • Topic Map Hier wird die Topic Map ausgewählt, die abgefragt werden soll.

  • Examples Ist die Auswahl-Box für die Beispiele.

  • Query Der Query der ausgeführt wird. Nach der Beispielauswahl wird das Textfeld mit dem entsprechenden TMQL-Query gesetzt. Diese kann jederzeit umgeschrieben.

  • Description Ist die Beschreibung des ausgewählten Beispiels.

Die Art der Abfrage, ob als GET- oder POST-Anfrage kann mittels der Auswahlknöpfe festgelegt werden. Vor allem um den selben Query mehrfach ausführen zu können empfiehlt sich die GET-Methode.

Es ist ebenfalls möglich zwischen dem puren JSON-Objekt oder einer HTML-Repräsentation auszuwählen. Das HTML-Ergenis listet alle Ergenistupel in einem Listenelement auf. Dabei wird jedes Element in der Ergebnisliste als <li>-Element ausgegeben. Enthält dieses wiederum ein oder mehrere Tupel werden diese nicht wieder getrennt. Das Ergebnis ist also eine Liste von JSON-Strings.
